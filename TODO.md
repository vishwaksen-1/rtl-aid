## Found during checklist.md review (2026-07-02)

- [x] rtllint `-I`/`--include` flag broken on Verilator 5.048 (regression, contradicted closed item above)
	Fixed: `_run_lint()` now builds `f"-I{d}"` (attached, no space) instead of `["-I", d]`. Confirmed
	the space-separated form genuinely fails on Verilator 5.048 (`%Error: Cannot find file containing
	module`), and the attached form works, via `tests/lint/test_verilator_integration.py`
	(verilator-gated). Unit coverage in `tests/lint/test_include_dirs.py`.

- [x] rtldoc doc-comment extraction — decided **won't do**, not a bug
	Discussed and settled: descriptions must stay hand-written. RTL comments are often multi-line,
	multi-style, or vendor-autogenerated (e.g. Vivado signature blocks) rather than documentation, so
	parsing them as doc source would silently produce wrong descriptions. `## Description` stays
	human-managed; new modules keep getting a `TODO: Add description` placeholder. No test needed —
	there's no bug to pin down.

- [x] rtllint warning-ID preservation (checklist items 12, 77 — partial; "discoverable rule set" as a
	CLI/docs feature is intentionally out of scope, see below)
	Fixed: `parse_lint_output()` now returns `{line: (warning_id, message)}` instead of a bare message
	string, and `tag_file()` renders it as `/* Check[WIDTHEXPAND]: ... */` (falls back to plain
	`/* Check: ... */` when verilator gives no dashed ID, e.g. a bare `%Error`). Covered in
	`tests/lint/test_parse_output.py` and `tests/lint/test_tag_file.py`.
	Not done: a `--list-rules`/enumerable rule-set surface — deferred, needs a product decision on what
	"rtl-aid's rule set" even means when 100% of it is delegated to verilator's `-Wall`.

- [x] rtllint sensitivity-list completeness (checklist item 24)
	Fixed via a new custom check — `check_sensitivity_completeness()` in `lint.py`, independent of
	verilator (verilator's `-Wall` doesn't catch this). Flags `always @(sig, ...)` blocks (explicit
	list, not `@*`, not clocked) that read a signal missing from their own list. Covered in
	`tests/lint/test_gaps.py`.

- [x] rtllint unlabeled `generate` blocks (checklist items 27, 75)
	Fixed via a new custom check — `check_unlabeled_generate()` in `lint.py`, independent of verilator.
	Both new checks run alongside the verilator pass in `rtllint`'s `main()`, merged into the same
	tag set (a verilator finding on a line always wins). Covered in `tests/lint/test_gaps.py`.

- [x] rtldoc port/parameter widths and types (checklist items 62, 65, 66)
	Fixed: `extract_module_and_ports()` now keeps the packed width (`data [7:0]`) or, for typedef'd
	ports (structs, enums), the type name (`p (pair_t)`, `st (state_t)`). Plain scalar ports
	(`input logic clk`) still render as just the name — no useful annotation to add. Covered in
	`tests/core/test_gaps.py`.

- [x] rtldoc parameter expression resolution (checklist item 63)
	Fixed, bounded scope: resolves simple integer arithmetic (`+ - * /`, parens, references to
	earlier parameters in the same module) via a small `ast`-based evaluator (no `eval`/`exec`).
	`DERIVED = BASE * 2` now renders as `DERIVED = BASE * 2  (= 8)` — raw expression kept alongside
	the resolved value rather than replacing it. Anything unresolvable (sized literals, unknown
	names, non-arithmetic) renders as `(unresolved)`, never a wrong silent guess. Covered in
	`tests/core/test_gaps.py`.

- [x] rtldoc `localparam` vs `parameter` distinction (checklist item 64)
	Fixed: `localparam` entries are now parsed and rendered in the Parameters list, suffixed
	`(localparam)` to distinguish them from real `parameter` overrides. Previously dropped entirely.
	Covered in `tests/core/test_gaps.py`.

- [x] rtldoc macro-driven port declarations (checklist item 57)
	Fixed, bounded scope: added `resolve_defines()`, a single-file `` `define `` substitution pass
	(no `` `include ``, no `` `ifdef ``) run before port extraction. A macro standing in for a
	direction keyword (`` `define IO_IN input``) no longer causes the port to silently vanish.
	Covered in `tests/core/test_gaps.py`.
